# V12 變更審核與結果視窗

## 一般使用者流程

1. Workflow 完成後只出現一個中央結果視窗。
2. 視窗可使用右上角、背景點擊或 `Escape` 關閉。
3. 在 Run Center 開啟「變更」。
4. 上方先看變更數量，中間選檔案，下方查看單一 Diff。
5. `auto_apply` 模式只需要閱讀，不會顯示不必要的套用動作。

## 進階使用者流程

1. 從「技術診斷」開啟「Patch 審核」。
2. 左側搜尋、勾選檔案。
3. 右側切換「並排」或「單欄」。
4. 左右與上下捲軸都只屬於 Diff 區，不會捲動整個 Drawer。
5. Review／Dry-run 模式才允許核准與選擇性套用。

## 版面原則

- Run Center 寬度不夠時不使用左右兩欄。
- 同一檔案不會同時出現在兩張重複卡片。
- Diff 預覽擁有固定可視區與獨立捲軸。
- 結果視窗、環境提示與技術診斷都能明確關閉。
